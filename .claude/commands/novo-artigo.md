Crie um novo artigo para o blog da Conecta Phone sobre o tema: $ARGUMENTS

Siga estas instruções exatamente:

## 1. Frontmatter

Gere o frontmatter YAML com os campos abaixo. A data de publicação deve ser a data de hoje.

- `title`: título atrativo e SEO-friendly em português, máximo 60 caracteres
- `description`: descrição de exatamente 150–160 caracteres para SEO e para o card do blog
- `pubDate`: data de hoje no formato YYYY-MM-DD
- `author`: "Equipe Conecta"
- `tags`: array de 3–5 tags relevantes (exemplos: "iPhone", "Bateria", "Reparo", "Samsung", "MacBook", "Emergência")

## 2. Conteúdo — mínimo 1.500 palavras

Escreva um artigo completo em português brasileiro com **no mínimo 1.500 palavras**. Para garantir isso, desenvolva cada seção com profundidade — não use respostas curtas.

### Estrutura obrigatória:

- **Introdução** (2–3 parágrafos, ~150 palavras): contextualize o problema, mostre que você entende a situação do leitor e apresente o que o artigo vai responder
- **4–6 seções com `##` headers**, cada uma com 200–300 palavras, cobrindo: sintomas detalhados, causas técnicas, o que fazer passo a passo, erros comuns, dicas de prevenção
- Use `###` subheaders dentro das seções maiores para organizar subtópicos
- Use listas numeradas e com marcadores, **negrito** para termos-chave, blockquotes para alertas importantes
- Tom técnico mas acessível, voltado a usuários comuns em Recife/PE
- Inclua dados concretos quando possível (porcentagens, prazos, nomes de modelos)
- **Seção final sobre a Conecta Phone** (~200 palavras): detalhe o serviço, prazo, garantia, diferenciais, e convide o leitor a entrar em contato

### SEO — obrigatório aplicar:

- Use a **palavra-chave principal** (derivada do tema) no título, nos primeiros 100 palavras do texto e em pelo menos 2 headers `##`
- Use **palavras-chave secundárias** relacionadas naturalmente ao longo do texto (ex: para "bateria iPhone": "autonomia", "carregar", "saúde da bateria", "ciclos de carga")
- Cada seção `##` deve ter um heading descritivo e específico — não use headings genéricos como "Introdução" ou "Conclusão"
- Inclua pelo menos um **link interno** para outro artigo do blog quando relevante, usando sintaxe markdown: `[texto do link](/blog/slug-do-artigo/)`
- Escreva parágrafos curtos (3–5 linhas), fáceis de escanear
- A description do frontmatter deve conter a palavra-chave principal e uma chamada para ação implícita

## 4. Nome do arquivo

Escolha um nome de arquivo em kebab-case baseado no título, curto e descritivo.

Exemplos:
- "Tela do iPhone trincada? Saiba o que fazer" → `tela-iphone-trincada.md`
- "Bateria do Samsung descarregando rápido" → `bateria-samsung-descarregando.md`

Salve em: `src/content/blog/<nome-do-arquivo>.md`

## 5. Ação final

Crie o arquivo em `src/content/blog/<nome-do-arquivo>.md` com todo o conteúdo gerado e confirme o caminho do arquivo criado.
